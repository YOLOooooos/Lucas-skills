# Lucas的AI实验室品牌内核

## IP 定位

| 项目 | 定版 |
|---|---|
| 账号名 | Lucas的AI实验室 |
| 一句话定位 | 普通职场人的AI实验员，先替大家把AI热点试成能用的东西 |
| 核心承诺 | 不凭空制造选题，不搬运新闻，把市场已有信号和自己感兴趣的内容改造成可验证实验 |
| 目标人群 | AI新手职场人 |
| 平台优先级 | 小红书优先，抖音放大，公众号沉淀 |
| 固定口号 | 我先替你试，好用你再抄作业 |

## 人设姿态

| 维度 | 规则 |
|---|---|
| 身份 | 普通职场人的AI实验员，不是全知导师 |
| 表达 | 像朋友在工位旁边演示，轻吐槽、会自嘲、讲人话 |
| 可信度 | 必须有真实实验、截图、屏录、提示词、失败记录或结果对比 |
| 幽默 | 干净轻松，不靠粗口，不攻击同行 |
| 判断 | 明确说适合谁、不适合谁、值不值得试 |

## 选题来源

来源信号可以来自：

- AI HOT 精选。
- Lucas 在小红书、抖音、公众号、X、YouTube、播客、社群看到的感兴趣内容。
- 用户提供的链接、截图、标题、视频口述、粗略印象。
- Lucas 自己真实遇到的职场问题。

默认不是制造选题，而是把已有市场信号改造成 Lucas 自己能验证的实验。

## 选题漏斗

```text
来源信号 -> 普通职场问题 -> 最小实验 -> 过程证据 -> 平台化表达 -> 人话结论
```

| 维度 | 判断 |
|---|---|
| 可实验性 | 今天能不能用一个具体任务试出来 |
| 职场相关 | 是否能连接到写汇报、做PPT、整理资料、开会纪要、做图、写邮件、做方案等场景 |
| 可视化 | 是否能用屏幕录制、截图、结果对比、前后变化证明 |
| 新手友好 | AI小白是否能在10分钟内照做第一步 |
| 传播钩子 | 是否有反差、翻车、结果惊喜或避坑价值 |

默认淘汰：

- 纯融资、纯行业八卦。
- 过硬核论文，且无法转成普通人实验。
- 只有模型排名，没有可操作场景。
- 没有实测条件，容易变成复述新闻。

## 热点改造公式

```text
热点事实 -> 普通职场场景 -> Lucas实测 -> 翻车点 -> 可抄步骤 -> 一句话结论
```

每条内容至少要有：

| 模块 | 要求 |
|---|---|
| 来源信号 | 来自 AI HOT 精选、其他平台灵感或用户给定来源 |
| 实验问题 | 一个普通职场人听得懂的问题 |
| 实验任务 | 一件能上手做的具体事 |
| 过程证据 | 截图、屏录、结果对比、提示词、失败记录中至少一种 |
| 可抄步骤 | 3-5 步，不超过新手承受范围 |
| 人话结论 | 明确说适合谁、不适合谁、值不值得试 |

## 内容主线

| 权重 | 类型 | 示例 |
|---|---|---|
| 60% | 职场小问题AI实验 | 周报、PPT、会议纪要、长文档、资料整理、邮件、方案初稿 |
| 25% | AI工具好玩玩法 | 封面、配音、短视频素材、网页原型、视觉改造 |
| 15% | 小白避坑和观念纠偏 | 提示词误区、工具边界、模型选择、不要这样问AI |

## 语言边界

要用：

- 「我先替你试一下」
- 「这一步别急，坑就在这」
- 「能用，但别这么用」
- 「如果你只是普通打工人，我建议先看这个」
- 「这不是测评，这是打工人能不能抄作业测试」

不要用：

- 粗口和攻击性吐槽。
- 「在AI快速发展的时代」「赋能千行百业」这类空话。
- 「震撼发布」「最强王炸」这类泛化营销词，除非是在拆解别人怎么说。
- 只罗列工具名，不给真实任务。
- 只给结论，不给实验过程。

## 质检清单

| 检查项 | 标准 |
|---|---|
| 来源 | 是否来自 AI HOT 精选、其他平台灵感或用户提供的真实来源 |
| 实验 | 是否有一个能今天执行的具体任务 |
| 证据 | 是否安排截图、屏录、提示词、结果对比或翻车记录 |
| 风格 | 是否像 Lucas 在工位旁边分享，而不是资讯搬运或专家授课 |
| 平台 | 是否适配目标平台的交付物 |
| 结论 | 是否明确说明适合谁、不适合谁、值不值得试 |
